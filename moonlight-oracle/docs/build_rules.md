\# Moonlight Oracle Build Rules



1\. Read all docs before making changes.

2\. Do not rewrite entire files unless necessary.

3\. Explain planned changes before changing files.

4\. Keep code modular and readable.

5\. Build only MVP features unless told otherwise.

6\. Use placeholder assets where needed.

7\. After each task, summarize:

&#x20;  - what was done

&#x20;  - what files changed

&#x20;  - what should be done next

8\. Do not break existing working features.

9\. Add basic verification/testing for each new system.

10\. Stop after completing the requested task.
